1. Executive Summary

On June 27, 2026, the Security Operations Center (SOC) detected anomalous activities originating from a production Linux server. An investigation was immediately launched following the standard incident response lifecycle. The analysis revealed that an external threat actor successfully compromised the root account via an SSH brute-force attack. Following the breach, the attacker executed a malicious payload and established a persistent outbound connection to a known malicious Command and Control (C2) server. Containment measures were swiftly enacted, and the system has been secured.

2. Incident Description & Evidence Collected

The investigation utilized several critical evidence sources to reconstruct the attack:

  • Authentication Logs (auth.log / system_logs.txt): Showed multiple failed SSH login attempts followed by a successful login.
  • Process Execution Logs: Revealed unauthorized use of wget and execution of a payload from the /tmp directory.
  • Network Traffic Logs (netstat / suspicious_connections.txt): Identified an active TCP connection to an external, untrusted IP address over port 443.

5. Containment Actions Taken

To immediately halt the attack and prevent further lateral movement, the following containment strategies were recommended and simulated:

  • Network Isolation: The affected server was logically isolated from the production network to sever the C2 connection to 203.0.113.50.
  • IP Blocking: The attacker IP 192.168.1.105 and C2 IP 203.0.113.50 were blocked at the perimeter firewall.
  • Process Termination: The malicious process sysupdate (PID 14601) was forcefully terminated using kill -9.
  • Account Suspension: The compromised root account password was immediately reset, and active sessions were killed. The rogue support_admin account was locked.

6. Remediation & Recovery Recommendations

To restore the system to a secure state and prevent future recurrence, the following remediation steps are mandated:

  1. Eradication: Remove the malicious file /tmp/sysupdate and delete the unauthorized support_admin user account completely.
  2. Hardening SSH: Disable root login over SSH (PermitRootLogin no in sshd_config) and enforce Key-Based Authentication rather than passwords.
  3. Implement Rate Limiting: Install and configure Fail2Ban to automatically block IP addresses exhibiting brute-force behavior.
  4. Vulnerability Patching: Perform a full system update to patch any existing OS vulnerabilities.
  5. Enhanced Logging: Forward all local authentication and process logs to a centralized SIEM (like ELK or Splunk) for real-time alerting.
