# Screenshots Evidence Checklist & Guide

Place all your screenshot evidence files in this folder using the standardized naming conventions listed below.

## Standardized Naming Convention
- `01_env_dvwa_dashboard.png` - Target DVWA setup and security level
- `02_burp_proxy_intercept.png` - Burp Suite HTTP proxy capturing requests
- `03_sqli_payload_execution.png` - Input field with `' OR 1=1 --`
- `04_sqli_data_dump.png` - Database records dumped on page
- `05_xss_reflected_popup.png` - Browser alert box executing `<script>alert('XSS')</script>`
- `06_xss_stored_guestbook.png` - Stored XSS persistent alert modal
- `07_auth_burp_intruder.png` - Burp Intruder brute-force attack window
- `08_session_devtools_cookies.png` - Browser DevTools inspecting missing `HttpOnly` / `Secure` flags
- `09_zap_scan_alerts.png` - OWASP ZAP Alerts dashboard summary

> Note: Screenshots should show full window contexts including URL address bars, tool titles, and timestamp evidence for maximal report credibility.
